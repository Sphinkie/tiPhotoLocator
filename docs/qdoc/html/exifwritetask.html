La classe ExifWriteTask permet d'écrire des metadata dans les photos JEG de façon asynchrone. More...

Detailed Description

* * *

Méthode par utilisation de QThreadPool.

Note: : les QRunnable n'héritent pas de QObject et ne peuvent donc pas communiquer avec les autres objets à l'aide de signaux.
Donc, à la fin du traitement, pour actualiser les données du PhotoModel, il faut faire un appel direct. Cependant, cela n'est pas contraire aux recommandations: mettre à jour des données se fait par appel synchrone.

[virtual] void ExifWriteTask::run()

Lancement de la tache. On lance exifTool dans un QProcess, et on écrit les metadata dans l'image JPG.
Cette tache est exécutée dans un thread QRunnable.
